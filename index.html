Lab 1

Introduction

During this lab, I explored two major tasks:

  1. Lab 1A: Familiarized myself with the SparkFun RedBoard Artemis Nano using the Arduino IDE.
  2. Lab 1B: Established Bluetooth communication between my Artemis board and a Python environment running in a Jupyter notebook.

By the end of these two lab sections, I set up my board, gathered sensor data, and used wireless connections to transmit information back and forth.


Parts Required
  • 1 × SparkFun RedBoard Artemis Nano
  • 1 × USB Cable

Lab 1A: Arduino IDE and Artemis Board

Objective:
1. Install and configure the Arduino IDE to program the Artemis board.
2. Upload example sketches to test the board’s built-in LED, serial communication, temperature sensor, and pulse-density microphone.

Prelab:
In preparation, I installed the Arduino IDE (latest version), added the SparkFun Apollo3 boards manager via the SparkFun JSON link, and checked that my board manager was up to date.

Tasks:

  1. I connected the Artemis board to my computer using the USB cable.
  2. I selected the correct board and port in the Arduino IDE.
  3. I uploaded the following example sketches from File -> Examples:
    • Blink – I confirmed the onboard LED was blinking.
    • Example4_Serial – I printed messages to the Serial Monitor and tested input/output.
    • Example2_analogRead – I monitored the onboard temperature sensor.
    • Example1_MicrophoneOutput – I tested the pulse density microphone.

Lab 1B: Bluetooth Connectivity

Objective:
1. Establish BLE (Bluetooth Low Energy) communication between my Artemis board and a Python environment in Jupyter Lab.
2. Learn how to send commands from my computer to the Artemis and receive data back from the board.

Prelab:
Before starting, I made sure I understood the provided Python code (in ble_python), which uses ArduinoBLE on the Artemis side. I noted that message sizes are limited to 150 bytes.

Setup and Configuration:

  1. I obtained my Artemis’s MAC Address by checking the serial output upon board startup and updated connections.yaml.
  2. I generated a new UUID to avoid collisions with classmates' boards and replaced it in both ble_arduino.ino and connections.yaml.
  3. I updated platform-specific settings as needed (e.g., if on Windows or Linux, set elif True: in base_ble.py if instructed).
  4. I uploaded the BLE sketch and confirmed that my Artemis board was advertising correctly.

Tasks:

  1. Test Basic BLE Communication: I verified that sending a PING command resulted in a PONG response.
  2. ECHO Command: I sent the string “HiHello” and confirmed I received “Robot says → HiHello :)”.
  3. SEND_THREE_FLOATS: I sent three floats (e.g., 1.23, 4.56, 7.89) and checked the Arduino’s serial output to confirm the correct values.
  4. GET_TIME_MILLIS: I requested millis() from my board and saw “T:xxxxxx” in my Python callback.
  5. Loop for 1s: I measured how frequently I could receive messages. This helped me estimate the data rate in messages per second or bytes per second.
  6. Store Timestamps in an Array: I added unsigned long timestamps[1000]; to collect data points, then used SEND_TIME_DATA to transmit them in bulk.
  7. Store Timestamps and Temperature: Similarly, I created parallel arrays for time and temperature, then used GET_TEMP_READINGS to send them to Python. I parsed these into lists and verified the results.
  8. Discussion of Two Methods: I compared streaming data in real time (each sample sent immediately) versus bulk transfer (storing an array then sending all data at once).

Conclusion

In Lab 1A, I installed the Arduino IDE and verified the basic functionalities of my Artemis board. In Lab 1B, I successfully set up a BLE link between my board and a Python environment, which allowed me to send and receive data wirelessly. This setup forms a strong foundation for my future labs, where I will likely collect and analyze sensor data, control motors, and debug without needing a wired connection.
